# Offer捕手技术方案说明书

## 一、问题诊断

这次作业选的是学生求职里的两个具体问题：第一，岗位太多，学生很难快速判断哪些机会和自己的专业、技能、城市偏好更接近；第二，看到心仪岗位后，不知道简历和要求差在哪里，也不知道先改哪一部分更容易通过初筛。

## 二、方案设计

Offer捕手做成网页Demo。学生先填写学历、学校、专业、毕业年份、目标城市、技能、项目经历和目标岗位。系统用本地岗位库做确定性匹配，输出岗位排名、匹配度、推荐档、命中技能和关键缺口。之后进入AI对话，学生可以继续问“这个岗位面试重点是什么”“简历应该怎么改”等问题。

这样设计是因为岗位排序不能完全交给大模型自由发挥，否则每次结果可能不一致。所以排序和分数由本地匹配引擎负责，AI主要负责解释和补充建议。

## 三、AI工具选型理由

Demo使用Streamlit开发，适合快速做可交互网页；AI接口采用OpenAI兼容格式，默认支持阿里云百炼/DashScope，也预留其他兼容模型配置。没有API Key时，系统仍能展示岗位匹配结果，只是AI问答会提示暂不可用。

## 四、关键配置

岗位数据包含50条校招/实习方向岗位，覆盖开发、产品、数据、设计、运营、HR等类型。匹配引擎看五个维度：技能匹配、学历门槛、专业相关度、目标岗位方向和城市偏好。技能匹配支持“Java”命中“熟悉Java或Go语言”，也能处理“本科及以上”等学历表达。

## 五、迭代记录

第一版只完成表单、岗位数据和AI推荐。后面发现只让AI写推荐理由不够稳定，所以增加了可解释匹配分数和推荐表。又补了密钥隔离，避免测试时读到本地真实API Key。最后增加多画像测试，验证后端、数据分析、设计、运营等画像能匹配到对应岗位族。

## 六、效果评估

当前Demo可以完成从填写资料、获得岗位推荐、查看简历缺口到继续追问AI建议的完整流程。系统测试共62项，覆盖配置读取、岗位匹配、页面渲染和主要工具函数。这个版本仍使用模拟岗位库，但已经能展示“先排序、再解释、再优化简历”的核心思路。
